# Gojica 项目设计汇总

## 📋 项目概述

**Gojica - 音乐人社区与服务平台**

一个服务于乐队、乐手及音乐爱好者的垂直社区平台，整合活动发布、乐手招募、排练房预约与二手设备交易四大核心功能。

---

## 🎯 技术架构

### 技术栈
- **客户端**：UniApp (Vue3 + TypeScript) - 微信小程序
- **服务端**：Node.js + Express
- **数据库**：MySQL 8.0
- **管理后台**：Vue3 + Element Plus

### 项目结构
```
Gojica/
├── miniprogram/              # 微信小程序客户端
├── server/                   # Node.js + Express 服务端
├── admin/                    # Vue3 管理后台
├── database/                 # 数据库脚本
```

## 🌐 接口规范

### 基础配置
- **Base URL**: `http://localhost:3000/api/v1`
- **认证方式**: JWT Bearer Token
- **字符编码**: UTF-8

### 响应格式
```json
{
  "code": 1000,
  "message": "操作成功",
  "data": {}
}
```

### HTTP 状态码
- 200: 成功
- 400: 请求参数错误
- 401: 未登录或Token过期
- 403: 权限不足
- 404: 资源不存在
- 500: 服务器内部错误

### 业务状态码
- 1000-1999: 成功类
- 2000-2999: 认证类
- 3000-3999: 客户端错误类
- 4000-4999: 服务器错误类

## 🔐 权限体系

### 用户身份
| 身份 | 说明 | 权限 |
|------|------|------|
| fan | 普通用户/爱好者 | 浏览、收藏、购买 |
| musician | 乐手 | 浏览、收藏、购买、出售商品、发布活动 |
| band | 乐队 | 浏览、收藏、购买、出售商品、发布活动、管理乐队 |
| venue | 场地商家 | 管理排练房、处理预约 |

### 角色
- user: 普通用户
- band_manager: 乐队管理者
- venue_owner: 场地所有者
- admin: 管理员
- super_admin: 超级管理员

### Token 配置
- Access Token 有效期: 24小时
- Refresh Token 有效期: 7天
- 算法: HS256

## 📁 文件存储规范

### 目录结构
```
/uploads
  /avatar        # 用户头像
  /band          # 乐队图片
  /activity      # 活动图片
  /room          # 排练房图片
  /product       # 商品图片
  /post          # 动态图片
  /audio         # 音频文件
  /video         # 视频文件
```

### 文件命名
```
{type}_{userId}_{timestamp}_{random}.{ext}

示例：
- avatar_1_1714214400000_abc123.jpg
- product_2_1714214400001_def456.png
```

### 限制
- 单文件大小: 5MB
- 支持格式: jpg, png, gif, webp, mp3, mp4

## 🛠️ 下一步开发计划

### 环境准备
1. 安装 Node.js 18+
2. 安装 MySQL 8.0（或使用已有数据库）
3. 安装 HBuilderX（UniApp 开发工具）
4. 安装 VSCode + Volar（TypeScript 支持）

### 项目初始化
1. 初始化 UniApp 小程序项目
2. 初始化 Node.js 服务端项目
3. 初始化 Vue3 管理后台项目
4. 配置数据库连接

### 快速开始
```bash
# 1. 导入数据库
mysql -u root -p gojica_db < database/gojica_db_init.sql

# 2. 启动服务端
cd server
npm install
npm run dev

# 3. 启动管理后台
cd admin
npm install
npm run dev
```
