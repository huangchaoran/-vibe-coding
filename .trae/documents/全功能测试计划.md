# Gojica 全功能测试计划

## 项目概述

Gojica 是一个音乐社区应用，包含：
- **后端**: Node.js + Express + Sequelize + MySQL (端口 3000)
- **前端**: 微信小程序 (UVUE) (HBuilderX 运行)

## 测试策略

### 1. 后端 API 测试

#### 1.1 单元测试 (Jest)
- 位置: `server/tests/unit/`
- 命令: `cd server && npm test`

#### 1.2 集成测试
- 位置: `server/tests/integration/`
- 覆盖模块:

#### 1.3 API 端点验证

| 模块 | 端点 | 方法 | 认证 | 测试要点 |
|-----|------|-----|------|---------|
| 首页 | `/api/v1/home/stats` | GET | 否 | 返回统计数据 |
| 首页 | `/api/v1/home/bands` | GET | 可选 | 返回乐队列表 |
| 首页 | `/api/v1/home/activities` | GET | 否 | 返回活动列表 |
| 认证 | `/api/v1/auth/login` | POST | 否 | 返回 token |
| 认证 | `/api/v1/auth/dev-login` | POST | 否 | 开发模式登录 |
| 乐队 | `/api/v1/bands` | GET | 可选 | 返回分页列表 |
| 乐队 | `/api/v1/bands/:id` | GET | 可选 | 返回详情 |
| 乐队 | `/api/v1/bands/:id/follow` | POST | 是 | 关注/取消关注 |
| 活动 | `/api/v1/activities` | GET | 否 | 返回活动列表 |
| 活动 | `/api/v1/activities/:id/signup` | POST | 是 | 活动报名 |

### 7. 常见问题排查

| 问题 | 可能原因 | 解决方案 |
|-----|---------|---------|
| 后端无响应 | 服务未启动 | `cd server && npm start` |
| 前端显示空数据 | API URL 不匹配 | 检查 api.uts 中的路径 |
| 登录失败 | token 生成问题 | 检查 jwt.js 配置 |
| 数据库连接失败 | MySQL 未启动 | 检查 MySQL 服务 |
| CORS 错误 | 跨域配置问题 | 检查 cors.js 配置 |
