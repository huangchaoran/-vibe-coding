### 2.1 修复 #1: UserService 的 toJSON() 问题

**问题描述**:
```javascript
// src/services/userService.js:20
const userJson = user.toJSON(); // TypeError: user.toJSON is not a function
```

**根本原因**:
单元测试中使用了 Mock 数据，没有正确模拟 Sequelize 实例的 `toJSON()` 方法。

**修复方案**:

#### 方案 A: 修复测试代码 (推荐)

**文件**: `tests/unit/services/userService.test.js`

**修复步骤**:
1. 修改测试文件中的 Mock 数据，确保包含 `toJSON()` 方法
2. 或者使用真实的 Sequelize 实例进行测试
3. 或者修改 Service 代码，使用更通用的数据序列化方式

**示例修复**:
```javascript
// 修复前
const mockUser = {
  id: 1,
  nickname: 'Test User',
  identity: 'fan'
};

// 修复后
const mockUser = {
  id: 1,
  nickname: 'Test User',
  identity: 'fan',
  toJSON: function() {
    return {
      id: this.id,
      nickname: this.nickname,
      identity: this.identity
    };
  }
};
```

#### 方案 B: 修改 Service 代码

**文件**: `src/services/userService.js`

**修复步骤**:
1. 检查数据来源
2. 如果是 Sequelize 实例，直接使用
3. 如果是普通对象，添加 toJSON 方法
4. 或者使用展开运算符 `{...user}`

**执行步骤**:
```
1. 读取 src/services/userService.js
2. 定位 getUserProfile, updateUserProfile, changeUserIdentity 方法
3. 修改数据序列化逻辑
4. 修改测试文件以匹配新的实现
5. 运行测试验证
```

**预计时间**: 30 分钟

### 2.2 修复 #2: Auth API 响应格式不统一

**问题描述**:
测试期望的响应格式与实际 API 返回格式不一致：

**实际返回**:
```json
{
  "accessToken": "...",
  "refreshToken": "...",
  "user": {...}
}
```

**测试期望**:
```json
{
  "token": "...",
  "refreshToken": "...",
  "userInfo": {...}
}
```

**修复方案**:

#### 方案 A: 修改 API 响应格式 (向后兼容)

**文件**: `src/controllers/authController.js`

**修复步骤**:
1. 定位 `login` 方法
2. 修改响应数据结构
3. 使用别名映射：
   - `accessToken` → `token`
   - `user` → `userInfo`

**示例修复**:
```javascript
// src/controllers/authController.js

// 修改 login 方法的响应
const responseData = {
  token: result.accessToken,      // 重命名 accessToken → token
  refreshToken: result.refreshToken,
  userInfo: result.user           // 重命名 user → userInfo
};

res.success(responseData, '登录成功');
```

**影响的测试文件**:
- `tests/integration/auth/auth.test.js`
- `tests/e2e/user-flow.test.js`

#### 方案 B: 更新测试期望 (推荐)

**文件**: `tests/integration/auth/auth.test.js`

**修复步骤**:
1. 读取当前测试文件
2. 更新期望的响应字段名
3. 从 `token` 改为 `accessToken`
4. 从 `userInfo` 改为 `user`

**执行步骤**:
```
1. 读取 tests/integration/auth/auth.test.js
2. 定位所有响应字段断言
3. 修改 expect(response.body.data).toHaveProperty('token') 
   → expect(response.body.data).toHaveProperty('accessToken')
4. 修改 expect(response.body.data).toHaveProperty('userInfo')
   → expect(response.body.data).toHaveProperty('user')
5. 同步更新 tests/e2e/user-flow.test.js
6. 运行测试验证
```

**预计时间**: 1 小时
