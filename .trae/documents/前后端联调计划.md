### 2.5 Token 认证检查 (P0)

#### 检查方法

```bash
# 无 Token 请求
curl http://localhost:3000/api/v1/users/me

# 带 Token 请求
curl -H "Authorization: Bearer <token>" \
  http://localhost:3000/api/v1/users/me
```

#### 检查要点

- [ ] **Token 存储** - `uni.setStorageSync('token', token)`
- [ ] **Token 读取** - `uni.getStorageSync('token')`
- [ ] **Token 传递** - `Authorization: Bearer ${token}`
- [ ] **Token 刷新** - refresh token 机制
- [ ] **Token 过期** - 401 错误处理

#### 前端 Token 处理

```typescript
// Gojica2.0前端/common/utils/api.uts

const request = (options: RequestOptions): Promise<any> => {
  const token = uni.getStorageSync('token')
  const headers = {
    'Content-Type': 'application/json',
    'Authorization': token ? 'Bearer ' + token : ''
  }
  
  // 401 错误处理
  if (res.statusCode === 401) {
    uni.removeStorageSync('token')
    uni.removeStorageSync('userInfo')
    uni.reLaunch({ url: '/pages/login/index' })
    reject(new Error('未授权，请重新登录'))
  }
}
```

#### 常见问题

| 问题 | 症状 | 解决方案 |
|------|------|---------|
| **Token 未传递** | 401 Unauthorized | 检查 Authorization Header |
| **Token 过期** | 401 Unauthorized | 实现 Token 刷新机制 |
| **Token 无效** | 401 Unauthorized | 检查 Token 生成和存储 |
| **页面未跳转** | 仍停留在受保护页面 | 实现 401 自动跳转 |

## 5. 问题诊断与修复

### 5.1 常见问题诊断

#### 问题 1: 404 Not Found

**症状**: 前端请求返回 404

**诊断步骤**:

```bash
# 1. 检查后端服务
curl http://localhost:3000/health

# 2. 检查路由定义
curl http://localhost:3000/api/v1/bands

# 3. 对比前端 API
# 文件: common/utils/api.uts
export const bandApi = {
  getList: () => apiGet('/bands')  # 确认路径一致
}
```

**解决方案**:

| 原因 | 修复方法 |
|------|---------|
| 后端未启动 | 启动后端: `cd server && npm start` |
| 路由路径错误 | 修正 `api.uts` 中的路径 |
| 路由未注册 | 检查 `src/routes/bands.js` |

#### 问题 2: 400 Bad Request

**症状**: 请求参数验证失败

**诊断步骤**:

```bash
# 1. 测试后端参数接收
curl -X POST http://localhost:3000/api/v1/bands \
  -H "Content-Type: application/json" \
  -d '{"name":"测试"}'  # 简化参数测试

# 2. 检查后端验证规则
# 文件: src/validators/bandValidator.js
```

**解决方案**:

| 原因 | 修复方法 |
|------|---------|
| 必填参数缺失 | 添加缺失参数 |
| 参数格式错误 | 修正参数格式 |
| 参数类型错误 | 确保类型正确 |

#### 问题 3: 401 Unauthorized

**症状**: 未授权访问

**诊断步骤**:

```bash
# 1. 检查 Token 是否传递
curl -v http://localhost:3000/api/v1/users/me

# 2. 检查 Token 格式
# 前端发送: Authorization: Bearer <token>
# 后端期望: Bearer token
```

**解决方案**:

| 原因 | 修复方法 |
|------|---------|
| Token 未存储 | 检查 `uni.setStorageSync` |
| Token 未读取 | 检查 `uni.getStorageSync` |
| Token 未传递 | 检查 `api.uts` Header 设置 |
| Token 过期 | 实现 Token 刷新 |

#### 问题 4: 数据字段不匹配

**症状**: 前端显示 undefined 或数据缺失

**诊断步骤**:

```bash
# 1. 获取后端返回
curl http://localhost:3000/api/v1/bands/1

# 2. 对比前端期望
# 前端: res.data.name
# 后端: { data: { name: "乐队名称" } }
```

**解决方案**:

| 原因 | 修复方法 |
|------|---------|
| 字段名不同 | 统一命名规范 |
| 嵌套结构不同 | 修正数据读取路径 |
| 类型不匹配 | 统一类型定义 |
