## 二、实现步骤

### 步骤1：获取微信 AppSecret

**操作人**: 用户（需要登录微信公众平台）

**步骤**:
1. 登录 [微信公众平台](https://mp.weixin.qq.com/)
2. 进入 **开发 → 开发设置**
3. 获取 **AppSecret**（需要管理员扫码）
4. 将 AppSecret 提供给开发人员

### 步骤2：配置后端环境变量

**文件**: `server/.env`

```diff
- WX_SECRET=your_wx_secret
+ WX_SECRET=真实的AppSecret值
```

### 步骤3：验证微信登录API

**测试命令**:
```bash
# 测试开发模式（当前可用）
curl -X POST http://localhost:3000/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"code": "dev", "identity": "fan"}'

# 测试真实微信登录（需要真实code）
curl -X POST http://localhost:3000/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"code": "微信返回的code", "identity": "fan"}'
```

### 步骤4：配置微信小程序服务器域名

**操作人**: 用户

**步骤**:
1. 登录微信公众平台
2. 进入 **开发 → 开发设置 → 服务器域名**
3. 添加 `https://your-domain.com` 到 **request合法域名**
4. 添加 `https://your-domain.com` 到 **uploadFile合法域名**

### 步骤5：测试完整登录流程

**测试流程**:
1. 打开微信开发者工具
2. 编译小程序
3. 进入「我的」页面
4. 选择身份并点击「微信授权登录」
5. 验证登录成功后跳转首页

## 四、风险与注意事项

### 1. AppSecret 安全
- AppSecret 是敏感信息，不应提交到代码仓库
- 应通过环境变量注入
- 生产环境应使用密钥管理服务

### 2. 微信开发者工具限制
- 游客模式下 `wx.login` 返回模拟数据
- 需要使用真实小程序账号测试
- 测试时需勾选「不校验合法域名」

### 3. 用户数据存储
- 用户首次登录会自动创建记录
- openid 作为唯一标识
- 支持用户身份切换（fan/musician/venue）

## 五、开发模式 vs 生产模式

| 模式 | 登录方式 | 适用场景 |
|------|---------|---------|
| 开发模式 | 自动生成模拟用户 | 本地开发、单元测试 |
| 生产模式 | 真实微信授权 | 正式发布、真机测试 |

当前后端已配置为：`NODE_ENV=development` 时自动使用开发模式登录。
