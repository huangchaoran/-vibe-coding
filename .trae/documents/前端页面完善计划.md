# Gojica 前端页面完善计划

## 目标
根据原型图（`Gojica前端\原型图\原型图详解.md` 和 `gojica-prototype.html`）完善前端页面，实现完整功能。

## 当前状态分析

### ✅ 已基本完成的页面
| 页面 | 状态 | 说明 |
|------|------|------|
| 首页 (index) | 80% | 布局完整，缺少点击跳转和API调用 |
| 乐队列表 | 85% | 功能完善，缺少创建入口跳转 |
| 个人中心 | 90% | 功能完整 |
| 发布动态 | 85% | 布局完整，缺少API调用 |

### ⚠️ 需要完善的页面

#### 1. 动态广场 (square/index.uvue) - 高优先级
| 问题 | 当前状态 | 目标 |
|------|----------|------|
| 分页加载 | `loadMore()` 为空函数 | 实现API分页加载 |
| 点赞功能 | 仅切换本地状态 | 调用 `postApi.like(id)` |
| 下拉刷新 | 无此功能 | 添加 `@refresherrefresh` |
| 图片展示 | 显示占位符 | 支持真实图片URL |

#### 2. 发布动态 (square/post.uvue) - 高优先级
| 问题 | 当前状态 | 目标 |
|------|----------|------|
| 发布功能 | 只显示Toast | 调用 `postApi.create()` |
| 图片上传 | 只能选图 | 实现图片上传到服务器 |

#### 3. 首页 (index/index.uvue) - 中优先级
| 问题 | 当前状态 | 目标 |
|------|----------|------|
| 搜索栏 | 仅样式 | 点击跳转搜索页 |
| 热门乐队 | 静态数据 | 点击跳转乐队详情 |
| 查看全部 | 无点击 | 跳转对应列表页 |
| 轮播图 | 静态 | 实现自动轮播 |

#### 4. 乐队模块 - 中优先级
- 乐队列表缺少创建乐队入口 (+ 按钮跳转)
- 乐队详情页需要完善成员展示、底部操作栏

#### 5. 活动模块 - 中优先级
- 活动列表需要完善报名状态显示
- 活动详情需要完善参与乐队展示

#### 6. 其他页面 - 低优先级
- 商品、招募、搜索页面功能基本完整
- 个人中心缺少身份管理入口

## 实施步骤

### Phase 1: 核心功能修复 (动态广场)
1. 修复 `loadMore()` 实现分页加载
2. 完善 `toggleLike()` 调用API
3. 添加下拉刷新功能
4. 优化图片展示

### Phase 2: 发布功能完善
1. 修改 `handlePost()` 调用API
2. 添加加载状态

### Phase 3: 交互功能完善
1. 首页添加点击跳转
2. 乐队列表添加创建入口
3. 各页面完善交互

### Phase 4: 其他页面优化
1. 活动详情完善
2. 个人中心完善
3. 代码质量检查
