# 首页底部导航栏优化计划

## 问题分析

### 当前实现的问题

| 问题 | 当前代码 | 影响 |
|------|----------|------|
| 滚动区域高度 | `height: calc(100vh - 60px)` | 固定计算，不够灵活 |
| 导航栏定位 | `position: absolute` | 可能被内容遮挡 |
| 内容区padding | `padding-bottom: 60px` | 固定值，与导航栏高度耦合 |
| 点击效果 | 无active状态切换 | 用户体验不佳 |

### 原型图要求

```
┌─────────────────────────────┐
│ 🏠   🎸   ✨   🛒   👤     │  ← 底部导航 60px
│ 首页  乐队  广场  市场  我的  │
└─────────────────────────────┘
```

- 5个Tab：首页、乐队、广场、市场、我的
- 选中状态：玫红色 `#e94560`
- 未选中：灰色 `#5a5a8a`

## 优化方案

### 1. 修复滚动区域被遮挡问题
```css
/* 当前 */
.content-scroll {
  height: calc(100vh - 60px);
}

/* 优化：使用flex布局 */
.index-container {
  display: flex;
  flex-direction: column;
  height: 100vh;
}

.content-scroll {
  flex: 1;
  overflow-y: auto;
}

.bottom-nav {
  flex-shrink: 0; /* 防止被压缩 */
}
```

### 2. 优化导航栏样式
```scss
.bottom-nav {
  // 移除 position: absolute
  // 改为 flex 布局的一部分
  display: flex;
  align-items: center;
  justify-content: space-around;
}
```

### 3. 添加点击反馈
```scss
.nav-item:active {
  opacity: 0.7;
}
```

## 实施步骤

### Step 1: 修改容器布局
- 将 `position: relative` 改为 Flex 布局
- 使用 `height: 100vh` + Flex 替代固定高度计算

### Step 2: 优化滚动区域
- 使用 `flex: 1` 让内容区自动填充剩余空间
- 确保 `overflow-y: auto`

### Step 3: 优化导航栏
- 移除 `position: absolute`
- 添加 `flex-shrink: 0` 防止被压缩
- 添加点击反馈效果

### Step 4: 同步到其他页面
需要同步修改的页面：
- `src/pages/band/list.uvue`
- `src/pages/square/index.uvue`
- `src/pages/product/list.uvue`
- `src/pages/user/index.uvue`
