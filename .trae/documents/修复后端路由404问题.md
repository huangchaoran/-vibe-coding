# 修复后端路由 404 问题计划

## 问题分析

### 根本原因

后端路由结构问题导致 `/api/v1/home` 返回 404：

```
前端请求: /api/v1/home          ❌ 404
后端实际: /api/v1/home/ (带末尾斜杠) ✅ 可以访问
```

### 路由挂载层级

```javascript
// app.js
app.use('/api/v1', require('./routes'))  // 第1层: /api/v1

// routes/index.js
router.use('/home', require('./home'))  // 第2层: /api/v1/home

// routes/home.js
router.get('/', asyncHandler(...))      // 第3层: /api/v1/home/ (需要末尾斜杠!)
```

### 问题

- 后端定义的路由: `/api/v1/home/` (带斜杠)
- 前端请求的路由: `/api/v1/home` (不带斜杠)
- Express 默认不自动重定向 `/home` 到 `/home/`

## 修复方案

### 方案 1: 修改后端路由支持无斜杠 (推荐)

在 `routes/index.js` 中添加无斜杠的路由别名：

```javascript
// routes/index.js
router.use('/home', require('./home'));
router.get('/home', (req, res, next) => {
  // 将 /home 重定向到 /home/
  req.url = '/';
  require('./home')(req, res, next);
});
```

### 方案 2: 使用 Express strict mode 配置

修改 Express 配置，允许无斜杠访问：

```javascript
// 在 routes/home.js 中
const homeRouter = express.Router({ strict: false });
```

### 方案 3: 修改前端请求地址 (不推荐)

将前端所有 `/api/v1/home` 改为 `/api/v1/home/`

## 实施步骤

### Step 1: 修复 home 路由

修改 `server/src/routes/index.js`，添加无斜杠支持：

```javascript
const express = require('express');
const router = express.Router();

router.use('/auth', require('./auth'));

// Home 路由 - 支持有斜杠和无斜杠
const homeRouter = express.Router();
homeRouter.get('/', require('../controllers/homeController').getHomeData);
homeRouter.get('/banners', require('../controllers/homeController').getBanners);
homeRouter.get('/stats', require('../controllers/homeController').getStats);
homeRouter.get('/bands', require('../controllers/homeController').getBands);
homeRouter.get('/activities', require('../controllers/homeController').getActivities);

router.use('/home', homeRouter);
router.get('/home', (req, res) => res.redirect('/api/v1/home/'));

router.use('/users', require('./users'));
router.use('/bands', require('./bands'));
router.use('/activities', require('./activities'));
router.use('/rooms', require('./rooms'));
router.use('/products', require('./products'));
router.use('/recruitments', require('./recruitments'));
router.use('/posts', require('./posts'));
router.use('/search', require('./search'));
router.use('/upload', require('./upload'));

module.exports = router;
```

### Step 2: 修复 users/me 路由

在 `server/src/routes/users.js` 中已添加 `/me` 路由，验证是否生效。

### Step 3: 测试验证

```bash
# 测试 /api/v1/home (无斜杠)
curl http://localhost:3000/api/v1/home

# 测试 /api/v1/home/ (有斜杠)
curl http://localhost:3000/api/v1/home/

# 测试 /api/v1/users/me (无 Token)
```
