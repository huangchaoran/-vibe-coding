### 4.3 数据完整性测试

```sql
-- 1. 用户数据完整性
SELECT COUNT(*) as total_users FROM users WHERE status = 1;
SELECT COUNT(*) as active_users FROM users WHERE last_login IS NOT NULL;

-- 2. 乐队数据完整性
SELECT COUNT(*) as total_bands FROM bands WHERE status = 1;
SELECT b.id, b.name, u.nickname as owner 
FROM bands b 
LEFT JOIN users u ON b.owner_id = u.id 
WHERE b.status = 1;

-- 3. 活动数据完整性
SELECT COUNT(*) as total_activities FROM activities WHERE status = 'recruiting';
SELECT a.id, a.title, u.nickname as organizer 
FROM activities a 
LEFT JOIN users u ON a.organizer_id = u.id;

-- 4. 关系完整性
SELECT COUNT(*) as orphan_band_members 
FROM band_members bm 
LEFT JOIN bands b ON bm.band_id = b.id 
WHERE b.id IS NULL;
```

## 5. 安全测试计划

### 5.1 认证与授权测试

| 测试项 | 测试方法 | 预期结果 | 状态 |
|--------|---------|---------|------|
| **Token 验证** | 访问需认证接口不带 Token | 返回 401 | ✅ 已测 |
| **Token 过期** | 使用过期 Token 访问接口 | 返回 401，提示刷新 Token | ✅ 已测 |
| **Token 伪造** | 使用伪造 Token 访问接口 | 返回 401 | ✅ 已测 |
| **权限控制** | 普通用户访问管理员接口 | 返回 403 | ⬜ 待测 |
| **越权访问** | 用户 A 访问用户 B 的私有数据 | 返回 403 或 404 | ⬜ 待测 |

### 5.2 输入验证测试

| 测试项 | 测试用例 | 预期结果 |
|--------|---------|---------|
| **SQL 注入** | `' OR '1'='1` | 数据不被注入，参数化查询有效 |
| **XSS 攻击** | `<script>alert(1)</script>` | 脚本不执行，特殊字符被转义 |
| **参数篡改** | 修改请求参数 ID | 返回 404 或 403 |
| **超长输入** | 输入超过字段最大长度 | 返回 422 验证错误 |
| **空输入** | 必填字段留空 | 返回 422 验证错误 |
| **格式错误** | 手机号格式错误、邮箱格式错误 | 返回 422 验证错误 |
