# 用户模块测试 - 产品需求文档

## 概述
- **摘要**: 对Gojica用户模块进行全面测试，包括前后端联调、API测试、数据流转验证和页面跳转测试
- **目的**: 确保用户模块功能完整、数据流转正确、用户体验流畅
- **目标用户**: Gojica平台用户（音乐爱好者、音乐人、场地商家、乐队）

## 目标
- [x] 用户身份认证与授权正常工作
- [x] 用户多身份管理功能完整
- [x] 用户个人中心页面功能正常
- [x] 前后端数据流转准确无误
- [x] 所有页面跳转流畅无问题
- [x] 用户模块API响应正确
- [x] 错误处理与用户提示友好

## 非目标（范围外）
- 不测试其他模块功能（乐队、活动等）
- 不进行性能和压力测试
- 不测试微信小程序原生功能

## 背景与上下文
- 后端已完成用户模块开发，支持用户多身份
- 前端已实现用户中心页面及子页面
- 之前测试发现部分API存在数据库表名不匹配问题，已修复
- 当前需要完整验证用户模块的功能正确性

## 功能需求

### 用户认证模块（auth）
- **FR1.1**: 微信登录功能正常（POST /api/v1/auth/login）
- **FR1.2**: 获取用户信息正常（GET /api/v1/users/profile）
- **FR1.3**: Token认证与刷新正常工作

### 用户信息模块
- **FR2.1**: 获取用户个人信息完整（GET /api/v1/users/profile）

- **FR2.3**: 绑定/添加身份功能正常（POST /api/v1/users/bind-identity）
- **FR2.4**: 移除身份功能正常（POST /api/v1/users/identities/remove）

### 用户数据模块
- **FR3.1**: 用户统计数据获取正常（GET /api/v1/users/stats）
- **FR3.2**: 我的活动列表获取正常（GET /api/v1/users/activities）

- **FR3.4**: 我的订单列表获取正常（GET /api/v1/users/orders）
- **FR3.5**: 我的收藏列表获取正常（GET /api/v1/users/favorites）
- **FR3.6**: 关注列表获取正常（GET /api/v1/users/follows）

### 页面跳转模块
- **FR4.1**: 从首页到用户中心跳转正常
- **FR4.2**: 用户中心到登录页跳转正常
- **FR4.3**: 用户中心到各子页面跳转正常

## 非功能需求
- **NFR1**: 所有API响应时间 < 1秒
- **NFR2**: 页面切换动画流畅，用户体验良好
- **NFR3**: 错误提示清晰友好
- **NFR4**: 数据展示准确，格式正确

## 约束
- **技术约束**: 使用UniApp + Vue3 + Node.js + MySQL
- **环境约束**: 开发环境测试，使用本地数据库
- **时间约束**: 单次测试周期不超过1天

## 假设
- [x] 后端服务正常运行在 http://localhost:3000
- [x] 数据库中已有测试数据
- [x] 前端项目已构建完成，可在微信开发者工具中运行

## 验收标准

### AC1: 用户认证流程
- **前提**: 用户打开小程序
- **动作**: 尝试登录并访问用户中心
- **预期结果**:
  - 登录成功，获取有效Token
  - 用户信息正确显示
  - Token持久化存储
- **验证方式**: programmatic
- **备注**: 需要测试登录和未登录状态

### AC2: 用户统计数据
- **前提**: 用户已登录
- **动作**: 访问用户中心
- **预期结果**:
  - 关注数、粉丝数、动态数、活动数正确显示
  - 数据与后端数据库一致
- **验证方式**: programmatic

### AC3: 页面跳转与权限
- **前提**: 用户处于不同状态（已登录/未登录）
- **动作**:
  - 点击菜单项尝试跳转
  - 返回上一页
  - Tab切换
- **预期结果**:
  - 未登录时跳转到登录页
  - 已登录时正常跳转到目标页面
  - 页面切换流畅无报错
- **验证方式**: human-judgment

### AC4: 子页面数据加载
- **前提**: 用户已登录，有测试数据
- **动作**: 依次访问我的活动、预约、订单、收藏
- **预期结果**:
  - 每个页面成功加载数据
  - 数据格式正确，显示正常
  - 空数据状态友好提示
- **验证方式**: human-judgment

### AC5: API响应格式
- **前提**: 后端服务运行
- **动作**: 调用用户模块各API
- **预期结果**:
  - 所有API返回200状态码
  - 响应JSON格式符合规范
  - 错误状态码处理正确
- **验证方式**: programmatic
