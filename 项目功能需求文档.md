## 🎯 项目概述

### 项目定位
**Gojica** 是一个面向音乐爱好者的社交平台，旨在连接音乐人、乐队、场地商家和普通音乐爱好者，打造一个完整的音乐生态圈。

### 核心价值
- **发现音乐**：浏览乐队、活动、招募信息
- **连接人脉**：关注乐队、加入社群
- **参与互动**：报名活动、发布动态、购买商品
- **多身份支持**：用户可以同时拥有多个身份（粉丝/音乐人/商家/乐队）

## 🎯 用户角色与权限

### 身份类型

| 身份 | 说明 | 权限 |
|------|------|------|
| `fan` | 普通粉丝 | 浏览、关注、报名、购买 |
| `musician` | 音乐人 | 发布动态、加入乐队 |
| `venue` | 场地商家 | 创建排练室、管理预约 |
| `band` | 乐队 | 创建活动、发布招募 |
| `admin` | 管理员 | 管理后台 |

### 多身份支持
用户可以同时拥有多个身份，例如：
- 一个用户可以同时是 `fan`（粉丝）和 `musician`（音乐人）
- 一个用户可以同时是 `musician`（音乐人）和 `venue`（商家）

## 🔒 安全规范

### 输入验证
- SQL 注入防护（Sequelize 参数化查询）
- XSS 防护（输出转义）
- CSRF 防护

### 敏感信息
- 密码加密存储（bcrypt，10轮）
- Token 仅在 Header 中传递
- 关键操作记录审计日志

### 限流策略
- 普通接口：60次/分钟
- 登录接口：5次/分钟
- 上传接口：10次/分钟
