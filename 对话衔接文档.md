# Gojica 2.0 项目对话衔接文档

## 项目概述

Gojica是一个音乐社区平台，包含后端API服务和前端小程序应用。

## 项目结构

```
Gojica2.0/
├── server/                    # 后端服务 (Express.js)
│   ├── src/
│   │   ├── controllers/       # 控制器
│   │   ├── models/            # 数据模型
│   │   ├── routes/            # 路由定义
│   │   ├── middlewares/        # 中间件
│   │   ├── services/          # 服务层
│   │   ├── validators/        # 请求验证
│   │   ├── utils/             # 工具函数
│   │   └── database/          # 数据库连接
│   ├── tests/                 # Jest测试
│   │   ├── integration/       # 集成测试
│   │   └── unit/              # 单元测试
│   └── .env                   # 环境配置
│
├── Gojica前端/                 # 前端应用 (uni-app)
│   ├── api/                   # API封装模块
│   ├── pages/                 # 页面组件
│   ├── store/                 # 状态管理
│   ├── utils/                 # 工具函数
│   └── pages.json             # 路由配置
│
└── 二次开发/docs/             # 开发文档
```

## 重要配置

### 后端环境变量 (.env)
```
NODE_ENV=development
PORT=3000
DB_NAME=gojica_test
DB_USER=root
DB_PASSWORD=66366888
JWT_SECRET=gojica_secret_key_2024_dev
```

### API响应格式
```json
{
  "code": 1000,    // 状态码
  "message": "操作成功",  // 消息
  "data": {}       // 数据
}
```

### 状态码规范
- 1000: 成功
- 1001: 创建成功
- 1002: 更新成功
- 1003: 删除成功
- 3001: 未授权
- 3002: 禁止访问
- 3003: 资源不存在
- 3004: 验证失败
- 4001: 服务器错误

3. **表结构**: 注意不同数据库的表结构可能不同（gojica_test vs gojica_db）
4. **用户认证**: 需要登录的接口需要先获取Token
